# Chapter 1: Ethereum as a State Machine

In it's most abstract form, Ethereum is a *deterministic state machine* -- a computer with a state (think memory) that changes in response to inputs, but in a deterministic way.

What do we mean by deterministic?  It means that for any given state and any given input, the resulting state is always the same.  Or more formally:

```
STF(S, I) -> S'
```

Where `S` is the current state, `I` is the input, and `S'` is the next state, and `STF` is the state transition function.

Therefore, to understand Ethereum in its abstract form, you really just need to understand three things:

- What does the state look like?
- What does the input look like?
- How does the state transition function actually operate?

## The state of Ethereum

The state of Ethereum is a combination of two things:

- The world state
- The history

## The input of Ethereum

The input of Ethereum is the block.

## The STF of Ethereum

The STF does two things:

- Update the world state by applying the block.
- Append the block to the history.
